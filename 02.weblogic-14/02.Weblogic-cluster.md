## WebLogic-14 Cluster Setup:


### WebLogic Cluster Node Requirements: 

#### 2 nodes (minimum recommended):

_Test/Dev setup:_
- Technically valid but not recommended for production
- Admin Server and Managed Server share the same machine
- No true fault tolerance

```
Node 1 → Admin Server + Managed Server (ms1)
Node 2 → Managed Server (ms2)
```


#### 3 nodes (recommended): 

_Most common production setup:_
- Admin Server is isolated
- Cluster has 2 Managed Servers for load balancing & failover
- If one Managed Server fails, the other handles traffic

```
Node 1 → Admin Server (dedicated)
Node 2 → Managed Server (ms1)
Node 3 → Managed Server (ms2)
```



### 3-Node WebLogic Cluster Overview: 

- **Admin Server** : 1 node required (does NOT serve application traffic). The Admin Server is NOT part of the cluster — it only manages it. 
- **Managed Servers** : minimum 2 to form a cluster (for failover)
- **Node Manager** : installed on every node that runs a Managed Server


| Node  | Role         | Hostname | IP        | Port | Components                   |
| ----- | ------------ | -------- | --------- | ---- | ---------------------------- |
| Node1 | Admin Node   | admin    | 10.1.1.10 | 7001 | AdminServer + NodeManager + 19c DB Client |
| Node2 | Managed Node | ms1      | 10.1.1.11 | 7003 | ManagedServer1 + NodeManager + 19c DB Client |
| Node3 | Managed Node | ms2      | 10.1.1.12 | 7003 | ManagedServer2 + NodeManager + 19c DB Client |



#### Cluster Architecture:

```
                        ┌─────────────────────────┐
         HTTP :80       │     LOAD BALANCER        │
  Users ──────────────► │   Nginx / Apache / OHS   │
                        └────────────┬────────────┘
                                     │
                    ┌────────────────┴───────────────┐
                    │                                │
         ┌──────────▼──────────┐       ┌─────────────▼──────────┐
         │  Node-2             │       │  Node-3                │
         │  10.1.1.11:7003     │       │  10.1.1.12:7003        │
         │  Managed Node-1     │◄─────►│  Managed Node-2        │
         │  [Cluster Member]   │       │  [Cluster Member]      │
         └──────────▲──────────┘       └─────────────▲──────────┘
                    │   Cluster Communication (Unicast)│
                    └──────────────┬───────────────────┘
                                   │ Management
                         ┌─────────▼──────────┐
                         │  Node-1 (admin)    │
                         │  10.1.1.10:7001    │
                         │  Admin Node        │
                         │  [Admin + NM]      │
                         └────────────────────┘
```


## Configure Load Balancer:

```
frontend weblogic_front
    bind *:80
    default_backend weblogic_cluster

backend weblogic_cluster
    balance roundrobin
    server ms1 192.168.1.11:7003 check
    server ms2 192.168.1.12:7003 check
```
